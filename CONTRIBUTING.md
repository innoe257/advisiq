# Contributing

This is a solo learning/portfolio project, but it follows a normal branch + PR workflow from Phase 2 onward:

1. Branch off `main` (`git checkout -b feat/short-description`).
2. Keep commits small and meaningful — no giant single commits.
3. Run `ruff check`, `mypy`, and `pytest` locally before opening a PR.
4. Open a PR into `main`; CI must pass before merging.

Never commit real credentials, institutional names, or non-synthetic data — see the constraints in the project brief.
